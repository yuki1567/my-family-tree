# デザインシステム

## 1. デザイン原則

### 1.1 基本方針

ユーザーが直感的に操作できるデザインシステム

**核心原則**:

- **階層の明確化**: 情報の重要度を視覚的に表現
- **空白の活用**: 適切な余白による見やすさの向上
- **一貫性**: 統一されたパターンによる予測可能性
- **実用性**: 美しさより使いやすさを優先
- **モバイルファースト**: レスポンシブデザインの採用
- **アクセシビリティ**: 色覚対応・色以外の情報伝達手段

### 1.2 対象ユーザー

- **メインユーザー**: 30-70 代の個人ユーザー
- **IT リテラシー**: 中程度
- **使用デバイス**: PC・タブレット・スマートフォン
- **使用シーン**: 家族の歴史記録・整理作業

## 2. カラーパレット

### 2.1 ブランドカラー

**プライマリカラー（オレンジ）**

```css
--color-primary: #f97316;
```

- **用途**: メインボタン、フォーカス状態、アクセント
- **使用例**: 「追加する」ボタン、設定ボタン、フローティングボタン

**セカンダリカラー（ブルー）**

```css
--color-secondary: #3b82f6;
```

- **用途**: 情報表示、リンク、補助的なアクション
- **使用例**: 情報アイコン、ヘルプリンク

**性別表現カラー**: 家系図での人物識別

**男性（ブルー系）**

```css
--color-male: #56b4f5;
```

- **用途**: 男性人物ノードの背景・ボーダー

**女性（ピンク系）**

```css
--color-female: #f1878c;
```

- **用途**: 女性人物ノードの背景・ボーダー

**性別不明（グレー系）**

```css
--color-unknown: #9ca3af;
```

- **用途**: 性別不明・未設定の人物ノード

### 2.3 機能カラー

**成功・完了状態**

```css
--color-success: #10b981;
--color-success-light: #d1fae5;
```

- **用途**: 保存完了、操作成功のフィードバック

**警告状態**

```css
--color-warning: #f59e0b;
--color-warning-light: #fef3c7;
```

- **用途**: 確認が必要な操作、注意喚起

**エラー状態**

```css
--color-error: #ef4444;
--color-error-light: #fee2e2;
```

- **用途**: 入力エラー、操作失敗のフィードバック

**情報表示**

```css
--color-info: #3b82f6;
--color-info-light: #dbeafe;
```

- **用途**: ヒント、説明、情報提供

### 2.4 ニュートラルカラー

**背景色**

```css
--color-background: #ffffff;
--color-background-secondary: #f8fafc;
--color-background-tertiary: #f1f5f9;
```

**テキスト色**

```css
--color-text: #1f2937; /* メインテキスト */
--color-text-secondary: #6b7280; /* 補助テキスト */
--color-text-light: #9ca3af; /* プレースホルダー等 */
```

**ボーダー色**

```css
--color-border: #e5e7eb; /* 標準ボーダー */
--color-border-light: #f3f4f6; /* 薄いボーダー */
--color-border-dark: #d1d5db; /* 濃いボーダー */
```

## 3. 家系図レイアウト仕様

### 3.1 レイアウト設計原則

**階層型レイアウト**

- **世代別配置**: 縦方向に世代を配置
- **水平整列**: 同世代内で水平配置
- **中央寄せ**: 全体を画面中央に配置

### 3.2 配置ルール

**間隔・距離**:

- **人物間隔**: 水平 160px、垂直 120px
- **世代間距離**: 120px（一定間隔を保持）
- **兄弟姉妹配置**: 近接配置でグループ化

**色分け**:

- **生物学的関係**: 緑系（#10b981）
- **養子縁組関係**: オレンジ系（#f97316）

### 3.3 レイアウト計算ルール

**世代計算**:

- **第0世代**: 親がいない人物（最上位）
- **第N世代**: 親の最大世代数 + 1

**配置調整**:

- **水平配置**: 同世代内で等間隔配置
- **垂直配置**: 上から下への階層表示
- **全体バランス**: 画面中央への配置、適切なマージン

## 4. タイポグラフィ

### 4.1 フォントファミリー

```css
/* システムフォント（高速・最適化済み） */
font-family:
  -apple-system, BlinkMacSystemFont, 'Segoe UI', 'Noto Sans JP',
  'Hiragino Kaku Gothic ProN', 'ヒラギノ角ゴ ProN W3', Meiryo, sans-serif;
```

**選定理由**:

- システムフォント優先による高速表示
- 日本語・英語の美しい表示
- デバイス間での一貫性確保

### 3.2 フォントサイズスケール

```css
--font-size-xs: 0.6rem; /* 補助情報、キャプション */
--font-size-sm: 0.8rem; /* 小さなラベル、説明文 */
--font-size-base: 1rem; /* 標準本文、フォーム */
--font-size-lg: 1.2rem;
```

### 3.3 フォントウェイト

```css
--font-weight-normal: 400; /* 標準本文 */
--font-weight-medium: 500; /* 強調テキスト、ラベル */
--font-weight-semibold: 600; /* 見出し */
```

### 3.4 行間（Line Height）

```css
--line-height-tight: 1.25; /* 見出し用 */
--line-height-normal: 1.5; /* 標準本文用 */
--line-height-relaxed: 1.625; /* 長文読み物用 */
```

## 9. レスポンシブ対応

### 9.1 ブレイクポイント

```
Breakpoints
├── mobile: ~767px
└── desktop: 768px~
```

### 9.2 適応的デザイン

```
Mobile First Approach
├── 基本: モバイル向けデザイン（~767px）
└── desktop: タブレット・PC最適化（768px~）
```
